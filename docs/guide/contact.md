# 联系与反馈

感谢你使用素流 (PureFlow)。无论是遇到问题、有功能建议，还是想和我们聊聊使用体验，都欢迎通过以下渠道联系我们。

## 反馈渠道

### GitHub Issues（推荐）

GitHub Issues 是我们首选的反馈渠道，适合提交 Bug 报告和功能建议：

- **地址**：[https://github.com/zhangjh/SuStream/issues](https://github.com/zhangjh/SuStream/issues)

**提交 Bug 报告时，建议包含以下信息：**

- 设备型号和 Android 版本
- 素流 (PureFlow) 版本号
- 问题的复现步骤
- 预期行为与实际行为的差异
- 相关截图或日志（如有）

**提交功能建议时，建议说明：**

- 你希望实现的功能
- 该功能解决的具体问题或使用场景
- 你期望的交互方式（如有想法）

## 反馈响应说明

我们重视每一条用户反馈，以下是反馈处理的一般流程：

1. **确认收到**：GitHub Issues 提交后会自动标记为待处理状态；邮件反馈我们会在 **1-3 个工作日** 内回复确认
2. **评估分类**：我们会对反馈进行分类（Bug、功能建议、使用咨询等），并评估优先级
3. **处理与跟进**：Bug 修复和功能开发会纳入版本迭代计划，你可以在 GitHub Issues 中跟踪处理进度
4. **版本发布**：修复和新功能会随版本更新发布，发布说明中会注明对应的 Issue 编号

::: tip 提示
通过 GitHub Issues 提交的反馈可以获得最快的响应和最透明的处理进度跟踪。我们建议优先使用这个渠道。
:::
